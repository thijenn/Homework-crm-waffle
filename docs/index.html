เปลี่ยนรหัสผ่านตัวเอง

ตั้งค่า

ทำอะไรได้

  • ผู้ใช้ที่ล็อกอินอยู่เปลี่ยนรหัสผ่านของตัวเองได้
  • ตรวจสอบครบ: รหัสเดิมถูกต้อง, ใหม่ ≥ 6 ตัว, ต้องไม่ซ้ำเดิม, ยืนยันต้องตรงกัน
  • ข้อความแจ้งเตือน 2 ภาษา (ไทย + อังกฤษ) · บันทึกลง Audit Log

วิธีทดสอบ

  • ไปเมนู ตั้งค่า → กรอกรหัสเดิม admin123
  • ลองกรอกรหัสใหม่ < 6 ตัว → ต้องขึ้น error
  • กรอกใหม่ถูกต้อง + ยืนยันตรงกัน → "เปลี่ยนรหัสผ่านสำเร็จ"
  • ออกจากระบบแล้วล็อกอินด้วยรหัสใหม่ → ต้องเข้าได้

💡 เบื้องหลัง & เหตุผล

รหัสผ่านไม่ถูกเก็บเป็น plaintext — ผ่านการ hash (FNV-1a 32-bit) ก่อนลง localStorage การยืนยัน "รหัสเดิม" จึงเทียบที่ค่า hash ไม่ใช่ตัวอักษรตรงๆ

กฎ "รหัสใหม่ต้องไม่ซ้ำของเดิม" + บันทึกเวลาเปลี่ยนและลง Audit Log ทุกครั้ง มีไว้ให้ผู้จัดการสอบย้อนได้ว่าใครเปลี่ยนรหัสเมื่อไหร่ · ข้อจำกัดที่รู้ตัว: FNV-1a เป็น hash เร็วเหมาะกับเดโม ระบบจริงควรย้ายไป bcrypt/argon2 ฝั่งเซิร์ฟเวอร์

Export รายชื่อลูกค้าเป็น CSV

รายชื่อลูกค้า

ทำอะไรได้

  • ส่งออกรายชื่อลูกค้า ตามที่กรองอยู่ เป็นไฟล์ CSV (UTF-8 + BOM เปิดภาษาไทยใน Excel ได้)
  • 17 คอลัมน์: แบรนด์, ชื่อ, เบอร์, วันที่, ผลโทร, ยอดฝาก, พนักงาน, นัดโทร, ห้ามโทร ฯลฯ
  • นำกลับเข้าระบบได้ (รูปแบบเดียวกับนำเข้า)

วิธีทดสอบ

  • เมนู รายชื่อลูกค้า → (เลือกฟิลเตอร์ เช่น แบรนด์ "มรกต")
  • กดปุ่ม ⬇️ Export CSV → ได้ไฟล์ตามจำนวนที่กรอง
  • เปิดไฟล์ใน Excel → ภาษาไทยอ่านได้ ครบทุกคอลัมน์

💡 เบื้องหลัง & เหตุผล

แก้ 2 จุดที่ CSV + Excel ภาษาไทยมักพังเสมอ: (1) ใส่ BOM EF BB BF นำหน้าไฟล์ Excel จึงตีความเป็น UTF-8 ภาษาไทยไม่กลายเป็นตัวต่างดาว · (2) เบอร์โทรเก็บเป็น "ข้อความ" ตลอด ไม่เคยถูกแปลงเป็นตัวเลข เลข 0 นำหน้าจึงไม่หาย

ใส่ quote แบบ RFC 4180 เฉพาะช่องที่มี , " \n และคอลัมน์ทั้ง 17 ตรงกับฟอร์แมต "นำเข้า" เป๊ะ → export แล้ว import กลับได้ครบ 100% ใช้เป็นไฟล์สำรองข้อมูลได้ในตัว

✅ ผ่านเกณฑ์ — round-trip export/import ตรงกัน 100%

ตัวกรองคิวโทรขั้นสูง

คิวโทร

ทำอะไรได้

  • กรองหลายเงื่อนไขพร้อมกันด้วย checkbox 3 กลุ่ม: ผลการโทร · สถานะลูกค้า · แบรนด์
  • สถานะ: คืนชีพแล้ว / รอติดตาม / เคยรับสาย / มีนัดโทรกลับ / ห้ามโทร
  • ปุ่ม ค้นหา + ออกรายงาน Excel + CSV ในหน้าเดียว

วิธีทดสอบ

  • เมนู คิวโทร → ติ๊กออก "คืนชีพแล้ว" เหลือ "รอติดตาม"
  • เลือกผลการโทรเฉพาะ "ไม่รับสาย" → กด 🔍 ค้นหา
  • ตารางแสดงเฉพาะรายที่ยังต้องตามต่อ · กด ออกรายงาน Excel

💡 เบื้องหลัง & เหตุผล

ตรรกะคือ "AND ระหว่างกลุ่ม · OR ภายในกลุ่ม" — เลือกหลายผลในกลุ่มเดียวเอามาทั้งหมด (OR) แต่ต้องผ่านทุกกลุ่มถึงโผล่ (AND) ตรงกับที่คนคิดจริง: "ไม่รับสาย หรือ ติดต่อไม่ได้ และ ต้องเป็นแบรนด์มรกต"

"สถานะลูกค้า" ไม่ได้เก็บเป็นฟิลด์ แต่ถอดสดจากข้อมูล (ยอดฝาก>0 = คืนชีพ · มีวันนัด = มีนัด · ติ๊ก dnc = ห้ามโทร) พนักงานจึงไม่ต้องมานั่งกรอกสถานะเอง คิวสะท้อนความจริงให้อัตโนมัติ

✅ ผ่านเกณฑ์ — ตรงตามแพทเทิร์นที่ขอ

รายงานรายสัปดาห์ / รายเดือน

รายงาน

ทำอะไรได้

  • สรุปผลงานอัตโนมัติแยกตามสัปดาห์ หรือ เดือน (สลับปุ่มได้)
  • คอลัมน์: โทร, รับสาย, รับสาย%, ไม่รับ, SMS, คืนชีพ, ยอดฝากกลับ, โบนัส, ROI
  • จัดกลุ่มจากวันที่โทรจริงในข้อมูล

วิธีทดสอบ

  • เมนู รายงาน → ดูตารางรายสัปดาห์ (สัปดาห์ 01/06, 08/06)
  • กดปุ่ม รายเดือน → สลับเป็นสรุปรายเดือน
  • ตรวจตัวเลข ROI / รับสาย% ว่าคำนวณถูก

💡 เบื้องหลัง & เหตุผล

จัดกลุ่มจาก "วันที่โทรจริง" ในข้อมูล ไม่ใช่วันนำเข้า — สัปดาห์ตัดแบบเริ่มวันจันทร์ (คำนวณจากเลขวันในสัปดาห์) ให้ตรงรอบงานของทีม ส่วนเดือนจัดด้วย key ปี-เดือน

ROI ไม่ใช่ตัวเลขลอยๆ แต่ = ยอดฝากกลับ ÷ โบนัสที่จ่าย ตอบตรงว่าโบนัส 1 บาทดึงเงินฝากกลับมากี่บาท · รับสาย% = รับสาย ÷ โทรทั้งหมด คู่กับ ไม่รับ% รวมกันได้ 100% เป็นจุดตรวจความถูกต้องในตัว

✅ ผ่านเกณฑ์ — bucketing ถูกต้อง

Export รายงานเป็น Excel

หลายหน้า

ทำอะไรได้

  • ออกไฟล์ .xls ที่เปิดใน Microsoft Excel / Google Sheets ได้ทันที (มีหัวตารางจัดรูปแบบ)
  • รองรับหลายหน้า: รายงาน, คิวโทร, ผลงานพนักงาน, Audit Log

วิธีทดสอบ

  • หน้า รายงาน → กด 📊 ออกรายงาน Excel
  • เปิดไฟล์ที่ดาวน์โหลด → เปิดใน Excel ได้ หัวตารางสีน้ำเงิน ข้อมูลครบ
  • ลองที่หน้า คิวโทร / ผลงานพนักงาน ด้วย

💡 เบื้องหลัง & เหตุผล

สร้างไฟล์ Excel โดยไม่พึ่งไลบรารีภายนอกเลย — เขียน <table> HTML แล้วส่งด้วย MIME application/vnd.ms-excel ซึ่ง Excel/Google Sheets เปิดได้เนทีฟ พร้อมหัวตารางจัดสีน้ำเงิน เลือกวิธีนี้เพราะเว็บเป็น static เปิดจากไฟล์ได้ทันทีไม่ต้องลงอะไร

ข้อแลกที่รู้ตัว: ได้ .xls แบบ HTML-table ไม่ใช่ .xlsx แท้ที่ฝังสูตรได้ — ถ้าต้องการสูตรจริงค่อยทำผ่านเซิร์ฟเวอร์ในเฟสถัดไป

หมายเหตุ: ไฟล์เป็น .xls (HTML-table) ซึ่ง Excel เปิดได้สมบูรณ์ · หากต้องการ .xlsx แท้พร้อมสูตร แนะนำทำผ่านเซิร์ฟเวอร์ในเฟสถัดไป
✅ ผ่านเกณฑ์ — เปิดใน Excel ได้จริง

นัดโทรกลับ (Callback)

นัดโทรกลับ

ทำอะไรได้

  • ตั้งวันนัดโทรกลับให้ลูกค้าแต่ละราย (ช่อง "นัดโทรกลับ" ในหน้าแก้ไข)
  • หน้า นัดโทรกลับ รวมรายการเรียงตามวันใกล้สุด พร้อมป้าย เลยกำหนด / ครบกำหนด / รออยู่

วิธีทดสอบ

  • หน้า รายชื่อลูกค้า → กด "แก้ไข" รายใดก็ได้ → ใส่วันนัด เช่น 15/06/2026 → บันทึก
  • ไปเมนู นัดโทรกลับ → ต้องเห็นรายนั้นพร้อมป้ายสถานะนัด

💡 เบื้องหลัง & เหตุผล

เปลี่ยน "ช่องวันที่" ธรรมดาให้กลายเป็น รายการงานจริง — เรียงวันใกล้สุดขึ้นก่อน แล้วติดป้ายความเร่งด่วนด้วยสี (เลยกำหนด=แดง · ครบกำหนดวันนี้=ส้ม · ยังรอ=น้ำเงิน)

จุดที่คิดเผื่อ: ป้ายสถานะเทียบกับ "วันล่าสุดในชุดข้อมูล" ไม่ใช่วันนี้จริงของเครื่อง — เดโมที่รันบนข้อมูลย้อนหลังจึงยังเห็นป้ายถูกต้อง ไม่กลายเป็นเลยกำหนดทั้งหมด

✅ ผ่านเกณฑ์ — บันทึก + แสดงผลถูกต้อง

บังคับสถานะห้ามโทร (Do-Not-Call)

DNC

ทำอะไรได้

  • ติ๊ก "ห้ามโทร" ในหน้าแก้ไข → บังคับกรอกเหตุผล → ติดป้ายแดง + บันทึก ประวัติเปลี่ยนสถานะ (ใคร/เมื่อไหร่/เหตุผล)
  • คิวโทร ตัดรายที่ห้ามโทรออกอัตโนมัติ · กรองดูเฉพาะ DNC ได้ · แจ้งหัวหน้าผ่าน Telegram
  • ปลดห้ามโทรได้ + มี log · ทุกการเปลี่ยนเข้า Audit Log

วิธีทดสอบ

  • แก้ไขลูกค้า → ติ๊ก 🚫 ห้ามโทร → บันทึก (ขึ้นป้ายแดง)
  • ไป คิวโทร → รายนั้นต้องหายจากคิว
  • หน้ารายชื่อ → ฟิลเตอร์ "🚫 ห้ามโทร" → เห็นเฉพาะรายที่ห้ามโทร

💡 เบื้องหลัง & เหตุผล

ออกแบบเชิง ความรับผิดชอบ / compliance — บังคับกรอกเหตุผลก่อนเซฟ (ปล่อยว่างไม่ได้) แล้วเขียนลง statusLog เป็นประวัติที่เก็บครบ ใคร/เมื่อไหร่/จากสถานะอะไรเป็นอะไร/เหตุผล จึงตอบได้เสมอว่าใครสั่งห้ามโทรเบอร์นี้และเพราะอะไร

การตัดออกจากคิวทำที่ ชั้นกรองคิว (queueFiltered) อัตโนมัติ ไม่ต้องลบลูกค้าทิ้ง กันโทรซ้ำโดยไม่ตั้งใจ + เด้งแจ้งหัวหน้าผ่าน Telegram ได้

✅ ผ่านเกณฑ์ — บังคับออกจากคิวได้จริง

แดชบอร์ดผลงานพนักงาน

วิเคราะห์

ทำอะไรได้

  • จัดกลุ่มตามชื่อ "พนักงานที่โทร" — วัดงานที่รับผิดชอบ, รับสาย%, %สำเร็จ, ยอดฝากที่ดึงกลับได้
  • เรียงตามยอดฝาก เห็นว่าใครปิดงานเก่งสุด · ออก Excel ได้

วิธีทดสอบ

  • แก้ไขลูกค้าหลายราย → ใส่ชื่อพนักงานในช่อง "พนักงานที่โทร" (เช่น สมชาย)
  • ไปเมนู ผลงานพนักงาน → เห็นแถวของสมชายพร้อมตัวเลขสรุป

💡 เบื้องหลัง & เหตุผล

จัดกลุ่มตามชื่อ "พนักงานที่โทร" แล้ววัดผลที่ผูกกับเงินจริง — "%สำเร็จ" = ลูกค้าที่ดึงกลับมาฝากได้ ÷ งานที่รับผิดชอบ ไม่ใช่แค่นับสายที่โทร จึงสะท้อนคนที่ "ปิดงานได้" ไม่ใช่แค่ "โทรเยอะ"

ซื่อตรงกับข้อมูล: ถ้ายังไม่มีการกรอกชื่อพนักงาน ระบบรวมเป็นกลุ่ม "(ไม่ระบุ)" ไม่สร้างชื่อปลอม — พอทีมเริ่มกรอกชื่อจริงก็แยกผลงานรายคนทันที

ข้อมูลจริงยังไม่มีชื่อพนักงาน จึงรวมเป็นกลุ่ม "(ไม่ระบุ)" — เมื่อทีมเริ่มกรอกชื่อ ระบบจะแยกผลงานรายคนทันที (ไม่ใช้ข้อมูลปลอม)

ทำอะไรได้

  • จัดกลุ่มลูกค้าตาม สัปดาห์ที่ถูกโทร แล้วเทียบว่ากลุ่มไหนดึงกลับมาฝากได้ดีกว่า (มีสีไล่ heat)
  • เทียบกลุ่มโปร 20% vs ไม่ได้รับ — กลับมาฝาก%, ฝากเฉลี่ย/คน, โบนัสจ่าย, ฝากสุทธิหลังหักโบนัส + สรุปบรรทัดเดียว "โปรคุ้มไหม"
  • ตอบคำถามเงินจริงของทีม: ติดตามแล้วได้ผลไหม โปรคุ้มไหม

วิธีทดสอบ

  • เมนู Cohort → ดูแต่ละแถว (สัปดาห์เริ่ม)
  • เทียบคอลัมน์ "อัตราคืนชีพ" — สีเขียว = กลุ่มที่ผลดี

💡 เบื้องหลัง & เหตุผล

ตอบคำถามเงินที่ "อัตรากลับมาฝาก" เพียวๆ ซ่อนไว้ — เทียบกลุ่มได้โปร vs ไม่ได้โปร แล้วคิด "ฝากสุทธิหลังหักโบนัส" (ยอดฝาก − โบนัสที่จ่าย) เพราะโปรอาจทำให้คนกลับมาเยอะแต่ขาดทุนหลังหักของแถม ตัวเลขสุทธินี้คือบรรทัดที่ฟ้องว่าโปรคุ้มจริงไหม

แบ่ง cohort ตามสัปดาห์ที่ถูกโทร (ตัดวันจันทร์เหมือนหน้ารายงาน) + ไล่สี heat (เขียว≥3% · ส้ม≥1.5% · แดงต่ำกว่า) ให้กวาดตาเห็นกลุ่มที่เวิร์กทันที

✅ ผ่านเกณฑ์ — แบ่ง cohort ตามวันจริง

Audit Log

ระบบ

ทำอะไรได้

  • บันทึกทุกการกระทำพร้อมเวลา + ชื่อผู้ใช้: เข้า/ออกระบบ, แก้ไข/เพิ่ม/ลบลูกค้า, นำเข้า, เปลี่ยนรหัส, เพิ่ม/ลบผู้ใช้, ส่ง SMS, export
  • ค้นหาได้ · ออก Excel · ล้าง log

วิธีทดสอบ

  • ไปเมนู Audit Log → เห็นรายการ "แก้ไขลูกค้า" พร้อมเวลาและชื่อ admin
  • ลองค้นหาคำว่า "เข้าสู่ระบบ"

💡 เบื้องหลัง & เหตุผล

ในระบบที่หลายคนใช้ร่วมกัน คำถามที่ต้องตอบได้คือ "ใครแก้ / ใครลบรายการนี้" — ทุก action จึงต่อท้ายเป็น record {เวลา, ผู้ใช้, การกระทำ, รายละเอียด}

ค้นหาแบบ substring ไม่สนตัวพิมพ์เล็กใหญ่ครอบทุกฟิลด์ และจำกัดเพดานที่ 1000 รายการล่าสุด กัน localStorage บวมไม่จำกัด

✅ ผ่านเกณฑ์ — บันทึกครบทุก action

คลังข้อความ SMS / ส่ง SMS

SMS

ทำอะไรได้

  • คลังข้อความ: เพิ่ม/ลบเทมเพลต ใช้ตัวแปร {name} {brand} {phone} แทนค่าอัตโนมัติ
  • เลือกกลุ่มผู้รับ (ยังไม่กลับมา/เคยรับสาย/คืนชีพ) + จำกัดแบรนด์ → ดูตัวอย่างข้อความจริง
  • กดส่ง → สร้างข้อความตามจำนวนผู้รับ + บันทึกลง Audit Log (ไม่รวม DNC)

วิธีทดสอบ

  • เมนู SMS → เพิ่มข้อความใหม่ที่มี {name}
  • เลือกเทมเพลต + กลุ่มผู้รับ → ดูตัวอย่างที่แทนค่าแล้ว + จำนวนผู้รับ
  • กดส่ง → ขึ้นสรุปจำนวน + ไปดูใน Audit Log

💡 เบื้องหลัง & เหตุผล

แทนค่าตัวแปร ({ชื่อ}/{เว็บ}/{เบอร์} รับทั้งรูปไทยและ {name}{brand}{phone}) เป็นรายคน แล้วโชว์ตัวอย่างข้อความจริง + จำนวนผู้รับก่อนส่ง ลดความเสี่ยงเทมเพลตผิดหลุดออกไปทีละพันคน

จุดสำคัญด้าน compliance: การคัดผู้รับ "ตัด DNC ออกทุกกลุ่ม" บังคับที่ชั้นเลือกผู้รับ (audience) เบอร์ห้ามโทรจึงไม่มีทางหลุดเข้า blast

การ "ส่งจริง" ต้องเชื่อม SMS Gateway ผ่านเซิร์ฟเวอร์ (เฟสถัดไป) — ปัจจุบันระบบสร้างข้อความ + บันทึกการส่งให้ตรวจสอบได้จริง
✅ ผ่านเกณฑ์ — เทมเพลต + แทนค่า + log ทำงาน

แจ้งเตือนผ่านบอท Telegram

แจ้งเตือน

ทำอะไรได้

  • ตั้งค่า Bot Token + Chat ID 2 กลุ่ม (ทีม/หัวหน้า) + เกณฑ์ยอดใหญ่/สายขั้นต่ำ + เปิดปิดรายประเภท
  • แจ้งทันที เมื่อมีเหตุการณ์: ยอดฝากใหญ่→ทีม, ปรับโบนัส→หัวหน้า, ตั้งห้ามโทร→หัวหน้า, นำเข้าข้อมูล→ทีม (เบอร์ลูกค้าถูก mask)
  • ส่งสรุป (แทน cron): รายวัน/รายสัปดาห์/เตือนพนักงานโทรน้อย + ปุ่มทดสอบส่ง
  • ส่งจริงผ่าน Telegram API · Telegram ล่มไม่ทำให้งานหลักพัง (try/catch + log)

วิธีทดสอบ

  • สร้างบอทกับ @BotFather → เอา Token ใส่ในหน้า แจ้งเตือน Telegram
  • ดึงบอทเข้ากลุ่ม → ใส่ Chat ID (กลุ่ม = เลขติดลบ) → กด ทดสอบส่ง
  • ปิด toggle ประเภทใด → ประเภทนั้นเงียบ · แก้ไขลูกค้าใส่ยอด ≥ เกณฑ์ → เด้งเข้ากลุ่มทีม

💡 เบื้องหลัง & เหตุผล

แยกปลายทาง 2 กลุ่ม (ทีม/หัวหน้า) + toggle เปิดปิดรายประเภท เพื่อกัน "alert fatigue" คนรับเฉพาะเรื่องที่เกี่ยวกับตัว · เบอร์ลูกค้าถูก mask เป็น 081-xxx-1234 ก่อนส่งเข้ากลุ่มแชต รักษาความเป็นส่วนตัว

จุดสำคัญสุด: ทุกการส่งห่อด้วย try/catch — Telegram ล่มหรือ Token ผิดจะแค่ warn ไม่ทำให้การบันทึกลูกค้า/งานหลักพัง (แจ้งเตือนเป็น best-effort ไม่ใช่ critical path) · สรุปรายวัน/สัปดาห์ใช้ปุ่มกดแทน cron เพราะ static web ไม่มีตัวตั้งเวลาฝั่งเซิร์ฟเวอร์

เว็บ static เก็บ Token ใน localStorage และส่งจาก browser · ระบบจริงต้องเก็บใน .env ฝั่งเซิร์ฟเวอร์ + ตั้ง cron ยิง /api/cron/* ตามไฟล์การบ้าน
✅ ผ่านเกณฑ์ — ส่งจริง + trigger + สรุป + mask เบอร์
UI

การปรับปรุง UI (6 จุด) + สิทธิ์ 3 ระดับ

🔐 สิทธิ์ 3 ระดับ (ADMIN/SUPERVISOR/AGENT): ล็อกเมนูตามสิทธิ์ — AGENT เห็นเฉพาะคิวโทร/ลูกค้า/SMS ของตัวเอง · หน้ารายงาน/วิเคราะห์/Telegram เฉพาะหัวหน้าขึ้นไป · Audit Log + จัดการผู้ใช้ เฉพาะ ADMIN (ทดสอบ: login เป็น agent1 แล้วเมนูจะหายไปตามสิทธิ์)

1. แถบเลือกช่วงวัน

วันนี้ / 7 วัน / เดือนนี้ / ทั้งหมด / กำหนดเอง — กรองทั้งแดชบอร์ดตามช่วงวัน พร้อมตัวเลขลูกค้าในช่วง

2. เทียบรายแบรนด์ + %

เพิ่มแถบสัดส่วน % ของยอดรวมในตาราง ดูเข้าใจง่ายขึ้น เรียงลำดับได้

3. แนวโน้มรายวันระดับโปร

พื้นที่ไล่สี + จุดสูงสุด + เส้นเฉลี่ยเคลื่อนที่ 7 วัน + สลับเมตริก ยอดฝาก/จำนวนคืนชีพ

4. เทียบรายเดือน

การ์ดสรุปต่อเดือน + ตารางเทียบ + Δ เปอร์เซ็นต์ขึ้น/ลง

5. ค้นหาด้วยชื่อ-นามสกุล

ช่องค้นหาเดียวค้นได้ทั้ง เบอร์โทร / ชื่อ / นามสกุล

6. เมนูจัดกลุ่ม + ปุ่มพิมพ์ PDF

แบ่งเมนูเป็นหมวด (ภาพรวม/วิเคราะห์/ปฏิบัติงาน/ระบบ) + ปุ่มพิมพ์ทุกหน้า

เอกสารนี้สร้างจากระบบจริง · ภาพถ่ายจากหน้าจอจริง · ทุกฟีเจอร์ทดสอบผ่าน (headless render + logic + ภาพจริง)
