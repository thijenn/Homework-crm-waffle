# CRM Dashboard — ระบบติดตามลูกค้าขาดฝาก

เว็บแดชบอร์ด CRM สำหรับทีมโทรติดตามลูกค้าที่หยุดฝาก — วิเคราะห์ผล, จัดคิวโทร, รายงาน และจัดการทีม
สร้างจากข้อมูลจริง (~9,100 ลูกค้า · 9 แบรนด์ · เดือนมิถุนายน 2026)

## เปิดใช้งาน
ดับเบิลคลิก `index.html` เปิดในเบราว์เซอร์ได้ทันที (ไม่ต้องต่อเน็ต ไม่ต้องลงโปรแกรม)

**ผู้ใช้ทดสอบ (3 สิทธิ์):**
| บทบาท | ชื่อผู้ใช้ / รหัสผ่าน | สิทธิ์ |
|---|---|---|
| ผู้จัดการ | `manager` / `manager1234` | เห็นทุกอย่าง + จัดการผู้ใช้ทุกระดับ |
| หัวหน้าทีม | `head1` / `head1234` | รายงาน/วิเคราะห์ + จัดการพนักงาน |
| พนักงานโทร | `admin` / `admin1234` | คิวโทร/ลูกค้า/SMS ของตัวเอง |

## ฟีเจอร์หลัก
- 📊 แดชบอร์ดภาพรวม + เลือกช่วงวัน (วันนี้/7วัน/เดือน/ทั้งหมด)
- 📈 วิเคราะห์: ผลการโทร, แนวโน้มรายวัน, เทียบแบรนด์/เดือน, ผลงานพนักงาน, Cohort
- ☎️ ปฏิบัติงาน: คิวโทรติดตาม (ตัวกรองขั้นสูง), นัดหมายโทรกลับ, รายชื่อลูกค้า, คลังข้อความ SMS
- 📋 รายงานรายสัปดาห์/รายเดือน + ออกรายงาน (Excel/CSV)
- 🔔 แจ้งเตือนผ่าน Telegram
- 🧾 ประวัติการใช้งาน (Audit Log)
- 👥 ระบบสิทธิ์ 3 ระดับ + กำหนดสิทธิ์เข้าถึงเมนูรายคน
- 🚫 สถานะห้ามโทร (Do-Not-Call) + ประวัติเปลี่ยนสถานะ

## โครงสร้างไฟล์
| ไฟล์ | หน้าที่ |
|---|---|
| `index.html` | หน้าเว็บหลัก |
| `app.js` | ตรรกะทั้งหมด (กราฟ/ตาราง/สิทธิ์/นำเข้า) |
| `styles.css` | ดีไซน์ (ธีมมืด) |
| `data.js` | ข้อมูลที่ดึงจาก Excel |
| `server.js` | เซิร์ฟเวอร์ทางเลือก (`node server.js`) |
| `docs/` | เอกสารฟีเจอร์ + ภาพประกอบ |

## หมายเหตุ
เวอร์ชันนี้เป็นเว็บ static (เก็บข้อมูล/ผู้ใช้ใน localStorage ของเบราว์เซอร์)
เหมาะกับการสาธิต/ใช้งานเดี่ยว — หากใช้หลายคนพร้อมกันควรต่อยอดเป็นเซิร์ฟเวอร์ + ฐานข้อมูล
